# Contributing to DocuMind

We'd love your help in making DocuMind better!

## Development Workflow
1. Fork the repository
2. Create a new branch (`git checkout -b feature/amazing-feature`)
3. Make your changes and commit them cleanly
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

## Areas for Contribution
- **Frontend Refinements:** CSS polish, accessibility, or mobile-responsiveness.
- **Backend Optimizations:** Adding new embedding models or parsing techniques (e.g. OCR for scanned PDFs).
- **Evaluating Pipelines:** Adding robust test suites using RAGAS.

Please format Python code with `black` and adhere to simple, readable React standard patterns.
